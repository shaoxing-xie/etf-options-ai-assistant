# 指数采集工具与原始接口说明

本文档说明各指数采集工具（含 `tool_fetch_index_opening`）如何调用原始数据接口，便于实现或对照其它指数工具的逻辑。

---

## 一、工具与 data_type 对应关系

| 工具名 | data_type | 说明 |
|--------|-----------|------|
| tool_fetch_index_realtime | realtime | 指数实时行情 |
| tool_fetch_index_historical | historical | 指数历史日/周/月 K 线 |
| tool_fetch_index_minute | minute | 指数分钟 K 线 |
| **tool_fetch_index_opening** | **opening** | **指数开盘数据（9:28 集合竞价）** |
| tool_fetch_global_index_spot | global_spot | 全球指数实时 |

统一入口：`tool_fetch_index_data(data_type=..., ...)`，别名通过 `tool_runner` 注入 `data_type`。

---

## 二、tool_fetch_index_opening：调用的原始接口

开盘数据逻辑与 `fetch_realtime` 类似：都是“当前快照”，只是时间点不同（开盘前后取到的“今开”即集合竞价结果）。

### 原系统实现位置

- **`src/data_collector.py`**
  - `fetch_index_opening_data()`：当日开盘数据
  - `fetch_index_opening_history()`：历史开盘（用于均值等，内部用日线）

### 原始接口调用顺序（已实现：新浪主 → 东财备）

| 优先级 | 接口 | 说明 |
|--------|------|------|
| 1 | **akshare `stock_zh_index_spot_sina()`** | 新浪指数实时/集合竞价，**主用**。返回列含 代码/名称/今开/昨收/涨跌幅，稳定可用。 |
| 2 | **akshare `stock_zh_index_spot_em(symbol=...)`** | 东方财富指数实时，**备用**。需轮询 symbol：`"沪深重要指数"`,`"上证系列指数"`,`"深证系列指数"`,`"中证系列指数"`，再按代码筛出目标指数（部分环境可能 RemoteDisconnected）。 |

### 字段映射（与 realtime 一致）

从 DataFrame 取列时兼容多种列名：

- **开盘价**：`今开` / `open` / `开盘` / `open_price`
- **昨收**：`昨收` / `close` / `close_yesterday` / `pre_close`
- **涨跌幅**：`涨跌幅` / `pct_chg` / `change_pct` / `涨跌%`
- **成交量**：`成交量` / `volume` / `vol` / `成交`
- **代码**：`代码` / `code` / `symbol`

### 默认指数列表（与原系统一致）

```python
index_codes = {
    "上证": "000001",
    "创业板": "399006",
    "深成指": "399001",
    "科创综指": "000688",
    "沪深300": "000300",
    "北证50": "899050"
}
```

实现时可按需改为 `index_codes: Optional[str] = None`（逗号分隔），再拆成列表/字典。

## 三、其它指数工具调用的原始接口（对照参考）

### 1. tool_fetch_index_realtime（fetch_realtime.py）

| 优先级 | 接口 | 说明 |
|--------|------|------|
| 1 | `ak.stock_zh_index_spot_sina()` | 新浪，主用 |
| 2 | `ak.stock_zh_index_spot_em(symbol=...)` | 东财，按 symbol 分类拉取后合并 |
| 降级 | 本地构造 `_get_fallback_data()` | 返回 0 与“数据暂时不可用” |

### 2. tool_fetch_index_historical（fetch_historical.py）

| 优先级 | 接口 | 说明 |
|--------|------|------|
| 1 | 原系统缓存 `get_cached_index_daily` + Tushare 备份 `fetch_index_daily_tushare` | 若可用 |
| 2 | Tushare `pro.index_daily(ts_code=..., start_date=..., end_date=...)` | 需 token |
| 3 | 新浪 HTTP `money.finance.sina.com.cn/.../CN_MarketData.getKLineData`，`scale=240`（日 K） | 直连 |
| 4 | `ak.index_zh_a_hist(symbol= Sina码, period="日k"/"周k"/"月k", ...)` | akshare |
| 5 | `ak.index_zh_a_hist_min_em(..., period="日k", ...)` | 东财日 K |

### 3. tool_fetch_index_minute（fetch_minute.py）

| 优先级 | 接口 | 说明 |
|--------|------|------|
| 1 | 新浪 HTTP `CN_MarketData.getKLineData`，`scale=1/5/15/30/60` | 主用 |
| 2 | 原系统缓存 `get_cached_index_minute`（若有） | 可选 |
| 3 | 东财/akshare 等备用 | 按实现 |

## 四、实现 tool_fetch_index_opening 时的建议

1. **与 realtime 对齐**  
   开盘数据本质是“开盘时刻的 spot”，接口与 realtime 相同（东财 + 新浪），只是语义为“开盘数据”，返回里可强调 `opening_price`、`timestamp` 等。

2. **交易日与 mode**  
   与其它指数工具一致：先做交易日检查（`check_trading_day_before_operation`），支持 `mode="test"` 跳过检查。

3. **返回格式**  
   与《工具参考手册》约定一致，例如：
   - `success`, `message`, `data`
   - `data`: 列表，每项含 `index_code`/`code`, `name`, `opening_price`, `pre_close`, `change`/`change_pct`, `timestamp` 等。

4. **原始接口调用顺序**  
   直接采用原系统顺序即可：先 **stock_zh_index_spot_em**（多 symbol 轮询），失败再 **stock_zh_index_spot_sina**；列名用上面“字段映射”做兼容。

5. **文件位置**  
   实现放在 `plugins/data_collection/index/fetch_opening.py`，对外提供 `fetch_index_opening()` 与 `tool_fetch_index_opening()`，由 `merged/fetch_index_data.py` 在 `data_type="opening"` 时导入并传入 `**kwargs`。

---

## 五、小结

- **tool_fetch_index_opening** 不单独再开新接口，而是复用与 **tool_fetch_index_realtime** 相同的数据源：**akshare 的 `stock_zh_index_spot_em`（主）和 `stock_zh_index_spot_sina`（备）**。
- 区别只在于业务含义（开盘/集合竞价）和返回字段命名（如 `opening_price`）；实现时可参考 `fetch_realtime.py` 的请求与列映射，再参考原系统 `src/data_collector.py` 的 `fetch_index_opening_data` 的默认指数列表与返回结构。
