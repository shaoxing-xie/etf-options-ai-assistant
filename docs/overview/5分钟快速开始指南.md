# 5分钟快速开始指南

**目标**：让新用户在 5 分钟内完成环境准备、插件安装和第一个工作流配置（面向 A股 / ETF 主场景）。

> 若用于生产部署，请优先执行 `docs/publish/README.md` 主线文档；本页定位为“本地快速验通”。

---

## 📋 前置条件检查

在开始之前，请确保您已满足以下条件：

- ✅ Windows 11 + WSL2 环境
- ✅ OpenClaw 已安装并运行（版本 2026.2.15 或更高）
- ✅ Python 3.8+ 已安装（在WSL中）
- ✅ 已进入项目目录：`/home/xie/etf-options-ai-assistant`（或你的实际仓库路径）

---

## 🚀 快速开始（5分钟）

### 步骤1：环境检查（30秒）

运行环境检查脚本，自动检查所有前置条件：

```bash
cd /home/xie/etf-options-ai-assistant
python3 check_environment.py
```

**预期输出**：
```
✅ Python版本检查通过
✅ OpenClaw路径检查通过
✅ 原系统路径检查通过
✅ 依赖包检查通过
✅ 所有检查通过，可以开始安装
```

如果检查失败，请根据提示修复问题。

### 步骤2：安装插件（1分钟）

运行自动安装脚本：

```bash
bash install_plugin.sh
```

**脚本会自动完成**：
- ✅ 创建OpenClaw插件目录
- ✅ 创建符号链接（无需复制文件）
- ✅ 创建插件配置文件
- ✅ 安装Python依赖包
- ✅ 注册插件到OpenClaw

**预期输出**：
```
✅ 插件目录创建成功
✅ 符号链接创建成功
✅ 插件配置文件创建成功
✅ Python依赖安装成功
✅ 插件注册成功
```

### 步骤3：验证安装（30秒）

检查插件是否成功注册：

```bash
# 检查OpenClaw状态
openclaw status

# 或者访问Dashboard
# http://127.0.0.1:18789/
```

在 Dashboard 中应该能看到 `option-trading-assistant` 插件（名称可后续调整），并能正常调用工具。

### 步骤4：测试第一个工具（1分钟）

在OpenClaw Dashboard中测试工具：

1. 打开 Dashboard：http://127.0.0.1:18789/
2. 找到项目插件（默认：`option-trading-assistant`）
3. 测试工具：`tool_check_trading_status`
   - 点击工具
   - 点击"执行"
   - 查看结果

**预期结果**：
```json
{
  "success": true,
  "data": {
    "status": "trading" 或 "closed",
    "is_trading_time": true 或 false,
    ...
  }
}
```

### 步骤5：配置第一个工作流（2分钟）

#### 5.1 使用示例工作流配置

复制示例工作流配置：

```bash
# 复制示例工作流到OpenClaw工作流目录
cp workflows/after_close_analysis_enhanced.yaml ~/.openclaw/workflows/
```

#### 5.2 在Dashboard中创建工作流

1. 打开 Dashboard：http://127.0.0.1:18789/
2. 进入"工作流"页面
3. 点击"创建新工作流"
4. 使用以下配置：

```yaml
name: test_workflow
description: 测试工作流
schedule: "*/5 * * * *"  # 每5分钟执行一次（用于测试）

steps:
  - name: check_status
    tool: tool_check_trading_status
    description: 检查交易状态
    params: {}
  
  - name: fetch_etf_data
    tool: tool_fetch_etf_realtime
    description: 获取ETF实时数据
    depends_on: [check_status]
    params:
      etf_code: "510300"
```

5. 保存并启用工作流

## 📝 示例配置

### 示例工作流：简单数据采集

```yaml
name: simple_data_collection
description: 简单数据采集工作流
schedule: "0 9 * * 1-5"  # 工作日 9:00

steps:
  - name: fetch_etf
    tool: tool_fetch_etf_realtime
    params:
      etf_code: "510300"
  
  - name: send_notification
    tool: tool_send_feishu_message
    depends_on: [fetch_etf]
    params:
      message: "ETF数据采集完成"
```
