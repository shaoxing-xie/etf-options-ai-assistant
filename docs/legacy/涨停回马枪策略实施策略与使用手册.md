# 涨停回马枪策略实施策略与使用手册

> 版本：v1.0（基于 2026-03 实施）  
> 适用系统：`etf-options-ai-assistant` + OpenClaw 研究模式一  
> 读者：后台开发 / 量化研究 / 运维同学

---

## 1. 策略概述

### 1.1 策略核心

- **交易对象**：A 股短线标的，聚焦「板块轮动 + 涨停回马枪」，只做有板块共振的**龙头股**，不做独立行情。
- **思想**：
  - 涨停只是“强势的结果”，持续性高度依赖**板块所处周期**：
    - 启动期：板块首日异动 → 次日大概率继续强势；
    - 发酵期：龙头连板后回调 → 二次上攻概率高；
    - 分歧期：龙头滞涨首阴 → 双底/回踩后的二次机会；
    - 退潮期：板块普跌 → 大概率继续下行。
  - 通过「板块热度评分 + 周期分型 + 龙头识别」先找到**板块+龙头**，再在不同周期上套用不同买点（次日低吸 / 3–5 日回调 / 双底）。

### 1.2 实施目标

1. **数据层**：每天盘后自动产出「涨停列表 + 板块热度 + 龙头打分」的结构化数据，留存 6–12 个月用于回测和复盘。
2. **回测层**：为「次日低吸」「3–5 日回调」「双底买入」三类核心策略提供统一的回测与参数敏感性框架。
3. **执行层**：在 OpenClaw 中提供**盘后自动流程 + 盘中参考节奏**，输出可直接执行的观察列表与风控建议。
4. **风控层**：统一仓位与风险规则，**不再采用“单标的=账户 2% 硬锁定”**，改为更灵活的策略/阶段仓位控制。

## 2. 模块与工具总览

### 2.1 主要 Python 模块

- `plugins/data_collection/limit_up/fetch_limit_up.py`
  - `tool_fetch_limit_up_stocks`：拉取指定日期或区间 A 股涨停池（基于 AkShare 东方财富接口）。

  - `tool_backtest_limit_up_pullback`：涨停回马枪回测工具（目前实现「次日低吸」核心策略）。

## 3. 数据管道与落盘规范（P0）

### 3.1 涨停股数据：tool_fetch_limit_up_stocks

**位置**：`plugins/data_collection/limit_up/fetch_limit_up.py`  
**TOOL_MAP 映射**：`"tool_fetch_limit_up_stocks": ("plugins.data_collection.limit_up.fetch_limit_up", "tool_fetch_limit_up_stocks")`

**入参（关键）**：
- `date: "YYYYMMDD"`：单日；
- `start_date, end_date: "YYYYMMDD"`：区间；
- `exclude_st: bool = true`：排除 ST；
- `exclude_tail_limit_up: bool = true`：排除 14:30 之后尾盘封板；

**核心字段**（每条记录）：
- `code, name`
- `limit_up_time`：首板/最后封板时间 → `HH:MM:SS`
- `change_pct`：当日涨跌幅 %
- `turnover_rate`：换手率 %
- `float_mv`：流通市值（元）
- `board_name`：所属行业/板块名称
- `continuous_limit_up_count`：连板数
- `latest_price, amount, date`

**使用示例**：

```bash
cd /home/xie/etf-options-ai-assistant
python3 tool_runner.py tool_fetch_limit_up_stocks '{"date":"20260306"}'
```

### 3.2 板块热度与周期：tool_sector_heat_score

**位置**：`plugins/data_collection/limit_up/sector_heat.py`

流程概念：
1. 从 `tool_fetch_limit_up_stocks` 获取当日涨停列表；
2. 按 `board_name` 聚合，得到每个板块的涨停数量与平均涨幅；
3. 调用 `tool_fetch_sector_data(sector_type="industry")` 获取板块层涨跌幅与净流入；
4. 计算热度评分 `score` 与周期 `phase`；
5. 按每个板块内龙头特征（连板数、封板时间、市值、换手率）选出 1–3 只龙头候选。

**输出结构（关键）**：

```json
{
  "success": true,
  "date": "YYYYMMDD",
  "sectors": [
    {
      "name": "电网设备",
      "score": 80,
      "limit_up_count": 6,
      "avg_change": 9.99,
      "net_flow": 123456789.0,
      "phase": "发酵",
      "max_continuous": 3,
      "leaders": [
        {
          "code": "000533",
          "name": "顺钠股份",
          "limit_up_time": "09:25:00",
          "continuous_limit_up_count": 3,
          "float_mv": 10191347712.0,
          "turnover_rate": 1.45,
          "leader_score": 80,
          "leader_reason": "3连板及以上；早盘封板；流通市值30-80亿；换手5-10%；板块内率先/领涨"
        }
      ]
    }
  ]
}
```

`phase` 取值：`启动 / 发酵 / 高潮 / 分歧 / 退潮`  
热度评分采用启发式组合（涨停数、涨幅、连板数、净流入等），支持后续替换为更精细的权重模型。

### 3.3 数据落盘：tool_write_limit_up_with_sector

**位置**：`plugins/data_collection/limit_up/daily_report.py`

功能：
- 调用 `tool_fetch_limit_up_stocks` + `tool_sector_heat_score`；

- 将结果写入：
  - `data/limit_up_research/YYYYMMDD_limit_up_with_sector.json`

JSON 结构：
- `date`
- `limit_up_list`：原始涨停列表（逐股）
- `sectors`：板块聚合与龙头信息（见上）

**使用示例**：

```bash
python3 tool_runner.py tool_write_limit_up_with_sector '{"date":"20260306"}'
```

## 4. 盘后自动流程与报告（P2）

### 4.1 一键盘后流程：tool_limit_up_daily_flow

**位置**：`plugins/data_collection/limit_up/daily_report.py`

签名：

```python
def tool_limit_up_daily_flow(
    date: Optional[str] = None,
    write_json: bool = True,
    write_report: bool = True,
    send_feishu: bool = False,
) -> Dict[str, Any]:
```

行为：
1. 若 `write_json=True`，调用 `tool_write_limit_up_with_sector` 落盘 JSON；
2. 读取对应 JSON，按板块热度与周期生成**次日观察列表 Markdown 报告**：
   - 路径：`data/limit_up_research/YYYYMMDD_report.md`
   - 内容包括：热点板块、热度、周期、推荐策略（次日低吸/3–5 日回调/双底）、板块内龙头列表；
3. 若 `send_feishu=True`，通过 `tool_send_feishu_notification` 以 `message` 形式推送简版报告正文（走统一通知渠道）。

**命令行示例**：

```bash
python3 tool_runner.py tool_limit_up_daily_flow '{"date":"20260306","write_json":true,"write_report":true,"send_feishu":false}'
```

返回示例：

```json
{
  "success": true,
  "date": "20260306",
  "json_path": "data/limit_up_research/20260306_limit_up_with_sector.json",
  "report_path": "data/limit_up_research/20260306_report.md",
  "sectors_count": 35
}
```

### 4.2 OpenClaw Cron 集成

在 `~/.openclaw/cron/jobs.json` 中新增任务：

- **ID**：`limitup-pullback-after-close`
- **时间**：`40 15 * * 1-5`（工作日 15:40）
- **行为**：在 `etf_analysis_agent` 的隔离会话中执行一轮 agentTurn，指示：
  - 调用 `tool_limit_up_daily_flow`，参数：`{"write_json": true, "write_report": true, "send_feishu": true}`
  - 产出 JSON + Markdown 报告并推送至通知渠道（若已配置飞书 webhook）

如需手动触发：

```bash
openclaw cron run limitup-pullback-after-close --timeout 1800000
```

## 5. 回测与参数敏感性（P1）

### 5.1 回测工具：tool_backtest_limit_up_pullback

**位置**：`plugins/backtest/limit_up_pullback.py`

功能（当前版本）：
- 从 `data/limit_up_research` 目录读取 `*_limit_up_with_sector.json`；
- 以日期排序，按 T-1/T 组合构造「次日低吸」机会；
- 调用 AkShare `stock_zh_a_hist` 获取标的日线（前复权），模拟如下逻辑：
  - 入场：次日开盘价（满足低开区间与量能条件时）；
  - 止损：跌破涨停日最低价或涨停价-止损阈值；
  - 止盈：到达目标涨幅（相对涨停价）；
  - 否则持有至持仓天数上限，到期按收盘平仓。

**工具签名**：

```python
def tool_backtest_limit_up_pullback(
    start_date: Optional[str] = None,
    end_date: Optional[str] = None,
    data_dir: Optional[str] = None,
    strategies: Optional[List[str]] = None,
    sector_score_min: int = 70,
    hold_days: int = 5,
    stop_loss_below_limit_pct: float = 5.0,
    target_pct: float = 5.0,
) -> Dict[str, Any]:
```

**返回**：
- `trades`：每笔虚拟交易（标的、入场/出场日期与价格、原因、收益等）；
- `stats`：胜率、平均盈亏、max drawdown、profit_factor、持仓天数分布等；
- `params`：实际使用的参数。

**使用示例**：

```bash
python3 tool_runner.py tool_backtest_limit_up_pullback '{"start_date":"20250101","end_date":"20251231"}'
```

> 注意：需要先保证对应日期范围内已有 `*_limit_up_with_sector.json` 数据文件。

### 5.2 参数敏感性：tool_backtest_limit_up_sensitivity

**位置**：同上文件。

核心思想：
- 对关键参数做网格搜索：
  - `sector_score_min`（热度阈值）
  - `hold_days`（持仓天数）
  - `stop_loss_below_limit_pct`（跌破涨停价止损幅度）
  - `target_pct`（相对涨停价目标涨幅）
- 对每组参数调用一次 `run_backtest`，统计胜率/平均盈亏/max drawdown 等；
- 计算一个综合得分：

> `score = 胜率*0.5 + max(0, 平均盈亏%)*0.3 - |最大回撤%|*0.2`

并选取得分最高的一组作为**当前推荐参数**（写入文档供参考）。

**工具签名**：

```python
def tool_backtest_limit_up_sensitivity(
    start_date: Optional[str] = None,
    end_date: Optional[str] = None,
    data_dir: Optional[str] = None,
) -> Dict[str, Any]:
```

## 6. 盘中执行节奏（供人工/半自动参考）

> 本节更多是执行建议，系统暂未完全自动化，只作为信号与执行的“前台说明书”，后台开发了解整体节奏即可。

- **9:15–9:30**：阅读前一日 `*_report.md` + 宏观/板块日报，锁定：
  - 板块热度 ≥ 70 的强热点；
  - 启动期/发酵期的龙头股（`leaders` 列表 + `leader_score` 高的一组）。
- **9:30–10:00**：重点观察龙头开盘形态与量能，确认是否满足「次日低吸」条件：
  - 低开 3–6%，量能不放大，板块整体不崩。
- **10:00–11:30**：
  - 按盘后观察列表逐一验证：价格区间 / 量能 / 板块强度；
  - 满足入场条件则按预设仓位与止损执行；
  - 不中则继续等待后续 3–5 日回调 / 双底机会。
- **13:30–14:30**：
  - 跟踪持仓盈亏、风控状态（是否触发止损/止盈）；
  - 不新开非计划内仓位。
- **14:50 以后**：
  - 原则上不再新开仓，只做必要的止损/止盈处理。

## 7. 仓位与风险控制（不再使用 2% 硬锁定）

### 7.1 仓位规则（策略层）

- 单策略单票：**5–8%** 账户资金（默认更稳健；极强行情可放宽，但需回测支持）。
- 总策略敞口：**≤15%** 账户资金（多个热点同时布局时的上限）。
- 不再采用“单标的=账户 2% 硬锁定”作为系统级强约束，硬限制只在以下情景显式启用：
  - 特殊风险事件演练；
  - 某些组合投资人要求的独立风险约束。

### 7.2 风控规则

- 单笔止损：
  - 跌破涨停日开盘价 -3%，或
  - 跌破涨停价 -5%（默认值，可通过参数敏感性优化）。
- 日内最大亏损（软阈值）：
  - 账户 **1.5–2%** → 停止当日所有新开仓，仅执行已有仓位的止损/止盈。
- 行为规则：
  - 连续 3 笔亏损：暂停策略，当日不再交易该类信号，盘后复盘并更新过滤条件/阈值。

### 7.3 系统仓位工具（供其他工作流调用）

- `tool_position_limit`（合并入口）：`plugins/merged/position_limit.py`
  - 默认 `apply_hard_limit=False`，`hard_limit_pct=None`，仅提供建议仓位；
  - 如需引入硬限制，必须**显式**传入 `apply_hard_limit=True` 且设置合理 `hard_limit_pct`。
- `tool_calculate_position_size` / `tool_check_position_limit` / `tool_apply_hard_limit`：  
  - 见 `plugins/analysis/etf_position_manager.py`，默认**不启用硬锁定**，仅在调用端指定时才会生效。

## 8. 运维与排错

### 8.1 常见路径

- 项目根：`/home/xie/etf-options-ai-assistant`
- 盘后数据：`data/limit_up_research/`
  - `YYYYMMDD_limit_up_with_sector.json`
  - `YYYYMMDD_report.md`
- OpenClaw workspace：`~/.openclaw/workspaces/etf-options-ai-assistant`
- OpenClaw 配置：`~/.openclaw/openclaw.json`
- OpenClaw 定时任务：`~/.openclaw/cron/jobs.json`

### 8.2 常见问题

- **板块数据报 UTF-8 BOM 错误**：
  - `tool_fetch_sector_data` 已在解析 JSONP 前 `lstrip("\ufeff")` 兼容 BOM；
  - 若仍失败，一般为东方财富接口临时异常，可降级为仅基于涨停列表计算板块热度（`net_flow=null`）。

- **monitor_510300 脚本推送失败**：
  - `scripts/monitor_510300_run.py` 现已对钉钉/飞书响应进行更严格校验：
    - 钉钉 `errcode != 0`、飞书 `code/StatusCode != 0` 会标记为 `success:false`；
  - 消息体过长会被截断为安全长度，避免被网关拒绝；
  - 若仍失败，可查看脚本输出中的 `error` 字段与网关返回体。

## 9. 后续扩展方向

1. **扩展更多买入形态**：在当前三大策略（次日低吸、3–5 日回调、双底买入）基础上，逐步加入「首阴买入」「突破确认」等形态，并在回测中统一比较。
2. **与 ETF 策略联动**：将板块热度与指数 ETF（如 510300）信号联动，用于择时或对冲。
3. **更精细的热度模型**：引入龙虎榜、北向资金、期权隐含波动率等维度，对当前基于行情+涨停数量的简化热度做升级。
4. **统一接入工作流 A（信号+风控巡检）**：将涨停回马枪信号纳入整体信号+风控巡检链路，在同一飞书报告中展示。

### 10.2 sector-rotation 技能封装（当前已落地部分）

**技能目标**：给定交易日，输出“热点板块 + 板块周期 + 龙头候选 + 建议策略类型（次日低吸/3–5 日回调/双底买入）”，并支持盘后自动落盘与报告。

**能力拆解与调用流程**：

1. **数据采集（涨停池与板块轮动）**  
   - 工具：
     - `tool_fetch_limit_up_stocks`：获取指定日期的涨停股列表；
     - `tool_fetch_sector_data`：获取行业层涨跌幅与净流入（数据源可用时）。
   - 行为：  
     - 对涨停股按 `board_name` 聚合，统计每个板块的涨停数量与平均涨幅；
     - 结合行业行情（若可用）作为热度评分的额外输入。

2. **板块热度评分与周期识别**  
   - 工具：`tool_sector_heat_score`  
   - 核心输出：  
     - 每个板块：`{name, score, limit_up_count, avg_change, net_flow, phase, leaders}`  
     - 其中 `phase ∈ {启动, 发酵, 高潮, 分歧, 退潮}`，`leaders` 为按连板数/封板时间/市值/换手率排序的 1–3 只龙头候选。

3. **结果落盘与次日观察列表**  
   - 工具：  
     - `tool_write_limit_up_with_sector`：写入 `data/limit_up_research/YYYYMMDD_limit_up_with_sector.json`；  
     - `tool_limit_up_daily_flow`：在前者基础上生成 Markdown 报告 `YYYYMMDD_report.md`，给出热点板块表格与建议策略类型。
   - 行为：  
     - 报告中对每个热点板块给出：热度分数、周期阶段、龙头名单（含连板数与 `leader_score`）、建议操作（次日低吸/3–5 日回调/双底/观望）。  
     - 该报告即是 sector-rotation 技能在盘后场景下的“人类可读视图”。

4. **命令行示例（单次技能调用）**：

```bash
# 1. 生成指定日期的数据与报告（完整 sector-rotation 流程）
python3 tool_runner.py tool_limit_up_daily_flow '{"date":"20260306","write_json":true,"write_report":true,"send_feishu":false}'

# 2. 仅获取结构化板块热度+龙头结果（供其他流程继续计算）
python3 tool_runner.py tool_sector_heat_score '{"date":"20260306"}'
```

### 10.3 技能视角下的研究模式一集成说明

在研究模式一（`~/.openclaw/prompts/research.md`）中，我们已经为“涨停回马枪”增加了策略 SKILL 段落。结合本手册，sector-rotation 技能在 Agent 中的推荐调用顺序为：

1. **盘后/非盘中场景**（回顾/研究某一交易日）：  
   - 优先调用 `tool_limit_up_daily_flow`（若当日数据尚未生成，内部会自动走 `tool_write_limit_up_with_sector`）；  
   - 在回答的「📂 数据与来源」部分，引用对应的 JSON/Markdown 文件路径；  
   - 在「📉 可执行建议 / 参数方案」中，利用 `phase` 与 `leaders` 字段给出策略类型建议（次日低吸/3–5 日回调/双底）。
2. **盘中场景**（用户问“今天有哪些板块/龙头可以做回马枪？”）：  
   - 若盘后数据已存在，可直接读取最近一个交易日的 `*_limit_up_with_sector.json` 与报告；  
   - 若用户明确指定“今日盘后请生成回马枪观察列表”，则建议在收盘后由 Cron 触发或人工运行 `tool_limit_up_daily_flow`。

### 10.4 技能级验收标准（设计/文档层）

为了保证后续技能扩展的可维护性，本手册下列出与 5 个技能相关的最低验收标准：

- **sector-rotation（当前唯一已落地技能）**：  
  - 每个交易日能稳定产出 `*_limit_up_with_sector.json` 与 `*_report.md`；  
  - 对任意热点板块，JSON 中至少包含：`score, phase, limit_up_count, leaders` 四类信息；  
  - 研究模式一下，Agent 能根据这些字段给出清晰的“板块周期 + 龙头 + 推荐策略类型”描述。
- **quantitative-screening**：已实现；多因子含 momentum / volatility / liquidity / **valuation**（来自 `tool_fetch_stock_financials`）；研究模式一与 `research.md` 已同步。
- **其余 3 个技能（dragon-tiger-list / capital-flow / northbound-flow）**：  
  - 在代码尚未落地阶段，文档中必须具备：

    - 与现有工具和数据源的依赖关系说明；  
    - 在技能分析报告中的落地路线与优先级规划。  
  - 每当某个技能进入“已实现”状态时，需要同步：  
    - 更新本节表格中的“当前状态”列；  
    - 在上方相关章节补充对应工具路径与调用示例；  
    - 更新 `~/.openclaw/prompts/research.md` 中的调用说明。
