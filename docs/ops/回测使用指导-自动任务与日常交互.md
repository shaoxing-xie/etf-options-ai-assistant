# 回测使用指导（`backtesting-trading-strategies` · 自动任务与日常交互）

本文档仅覆盖 Skill **`backtesting-trading-strategies`**：通过仓库内脚本做宽基 ETF / 加密等标的的技术指标回测与参数网格，**不包含**涨停回马枪专题工具（`tool_backtest_limit_up_*`）的使用说明。

**相关代码与配置**

- 脚本目录：`skills/backtesting-trading-strategies/scripts/`（`backtest.py`、`fetch_data.py`、`optimize.py` 等）
- 统一配置：`skills/backtesting-trading-strategies/config/settings.yaml`（由 `scripts/skill_settings.py` 加载）
- **OpenClaw 自动识别（须挂载 Skill 并重载 Gateway）**：`skills/ota-backtesting-integration-brief/SKILL.md` 规定 **分支 A = 默认单条 `exec` 跑脚本**（禁止先串 `tool_fetch_*` / 指标工具）；仅 **分支 B**（用户明确要求与 58 指标/缓存对账）才先走 MCP 数据链。`etf_main` / `etf_analysis_agent` 等应在 `openclaw.json` 中挂载该 Skill（见 `config/snippets/openclaw_agents_ota_skills.json`）。**`etf_cron_research_agent`** 的 `skills` 由 `config/agents/cron_agents.yaml` 的 `requiredSkills` **全量定义**（已含 `ota_backtesting_integration_brief` + `backtesting-trading-strategies`）；合并后须 `python scripts/render_agents_config.py` 并重启 Gateway。
- 按需工作流（与上同序）：`workflows/backtesting_research_on_demand.yaml`

---

## 1. 能力与边界

- **策略**：内置 8 种（SMA/EMA 交叉、RSI、MACD、布林带、突破、均值回归、动量等），见 `scripts/strategies.py`。
- **标的**：六代码 A 股/ETF（如 `510300`、`510300.SS`）默认走本仓库 **`plugins/data_collection`** 多源日线（与 `openclaw-data-china-stock` 同源策略）；美股/加密等可用 **`--data-source yfinance`** 或 YAML 中 `data.provider: coingecko`。
- **产出**：`skills/backtesting-trading-strategies/reports/` 下 `*_summary.txt`、`*_trades.csv`、`*_equity.csv`、`*_chart.png`（若装了 matplotlib）；`optimize.py` 另写 `optimization_*.csv`。
- **局限**：脚本侧为研究级回测引擎，**不**替代盘中风控与实盘执行链路；长周期策略（如 SMA 默认 `lookback=200`）需要足够历史 K 线，短窗口会直接报错退出。

## 2. 环境与配置

### 2.1 依赖与网络

- 依赖见 `skills/backtesting-trading-strategies/SKILL.md`（`pandas`、`numpy`、`yfinance`；CN 场景建议 `akshare` 等）。
- **首次**拉取日线通常需外网；命中 `skills/.../data/*_1d.csv` 缓存后可离线复跑。

### 2.2 `settings.yaml` 与环境变量

- **`data.provider`**：`auto` | `china` | `yfinance` | `coingecko`（CLI 未指定 `--data-source` / `--source` 且未设 `BACKTEST_DATA_SOURCE` 时使用）。
- **`data.cache_dir`**、**`reporting.output_dir`**：相对 Skill 根目录的路径，除非写成绝对路径。
- **`BACKTEST_SKILL_SETTINGS`**：指向另一份同结构 YAML，覆盖默认配置文件路径。
- **数据源优先级**：`--data-source` / `--source`（CLI）→ 环境变量 **`BACKTEST_DATA_SOURCE`** → YAML **`data.provider`** → `auto`。
- CN 场景若本地管道失败且**执意**走 Yahoo 兜底（不推荐）：**`BACKTEST_CN_FALLBACK_YFINANCE=1`**。

## 3. 命令参考（日常与 `exec`）

### 3.1 常用命令（路径请按部署机器改为绝对路径）

```bash
# 列出策略（无需 --strategy）
python3 /home/xie/etf-options-ai-assistant/skills/backtesting-trading-strategies/scripts/backtest.py --list

# 沪深300ETF + RSI（默认数据路径：plugins/data_collection）
python3 .../scripts/backtest.py --strategy rsi_reversal --symbol 510300.SS --period 1y --quiet

# 显式 Yahoo（加密等）
python3 .../scripts/backtest.py --strategy rsi_reversal --symbol BTC-USD --period 120d --data-source yfinance --quiet

# 仅拉数（默认 YAML：六代码 + 1d → 本地管道）
python3 .../scripts/fetch_data.py --symbol 510300.SS --period 120d

# 小网格优化
python3 .../scripts/optimize.py --strategy rsi_reversal --symbol 510300.SS --period 1y --param-grid '{"period":[12,14]}'
```

### 3.2 仓库入口（便于 OpenClaw `exec`：单条 `python3`、无 `cd &&`）

```text
python3 /home/xie/etf-options-ai-assistant/scripts/run_backtest_trading_strategies.py --strategy sma_crossover --symbol 510300.SS --period 1y --params '{"fast_period":5,"slow_period":10}'
```

与直接调用 `skills/.../scripts/backtest.py` 等价，仅 argv 转发。

### 3.3 标的写法与缓存

- 缓存文件名：`{symbol 将 / 换成 _、将 - 换成 _}_1d.csv`（例如 `510300.SS` 与 `510300` 为**两份** CSV）。
- 团队内对同一标的建议**统一**一种代码写法，避免读错缓存。

### 3.4 高 `lookback` 策略与窗口

- `sma_crossover` / `ema_crossover` 等 `lookback=200`，脚本要求至少 **`max(50, lookback+5)`** 根 K 线。
- 短 `--period` 会**按设计失败**（exit 1）；需加长 `--period` 或使用 `--start` / `--end` 覆盖足够日历区间。

## 4. 自动任务（Cron / Workflow）

### 4.1 适用场景

- 收盘后固定窗口复跑同一策略、落盘 `reports/` 供对比；
- 周期性参数小网格（控制 `param-grid` 规模，避免超时）。

### 4.2 `jobs.json` 提示词示例（单条工具等价：只 spawn 一次解释器）

```text
执行 backtesting-trading-strategies 回测（生产巡检）。
第一条输出必须为单条 shell 等价命令，例如：
python3 /home/xie/etf-options-ai-assistant/skills/backtesting-trading-strategies/scripts/backtest.py --strategy rsi_reversal --symbol 510300.SS --period 1y --quiet
禁止 cd && 与复合管道。完成后仅输出：退出码含义、reports 下新生成文件名摘要、若失败则 stderr 最后一行。
```

### 4.3 验收建议

- 进程 **exit 0**；
- `reports/`（及必要时 `data/`）出现预期新文件；
- 失败时保留**可复现**完整命令与错误末行，避免静默成功。

## 5. 钉钉 / OpenClaw 日常交互（`backtesting-trading-strategies`）

下文「**仓库根**」请替换为部署机上的绝对路径，例如：`/home/xie/etf-options-ai-assistant`。

### 5.1 通道约束（必守）

- **`exec` 前置**：只允许**一条** `python3 /绝对路径/...py ...`，**禁止** `cd ... &&`、`bash -c` 多段、管道重定向套娃（否则会触发 `exec preflight: complex interpreter invocation`）。
- **解释器**：环境只有 `python3` 时**不要**写 `python`。
- **输出**：钉钉/OpenClaw 易触顶 **`length`**（回复被截断、半句英文）；要求模型 **全文控制在约 400 字内**，或拆成**两条消息**（先结论+指标，第二条风险与下一步命令）。
- **数据**：六代码 ETF 默认走本地 `plugins/data_collection`，**不要**用 `web_search` 代替拉数；无结果时根据 stderr 改 `--period` / `--data-source` / 配置，**勿编造**收益数字。

### 5.2 研究员 @ 机器人（自然语言，可复制）

以下可直接发在钉钉对话里；Agent 应翻译成 **§5.4** 的单条 `exec`，再按 **§5.3** 摘要回群。

- 「用 **RSI 反转** 给 **510300** 做最近 **一年** 回测，标的研究级；回完告诉我总收益、最大回撤、交易次数，并说明用的数据源（`price_loader`）。」
- 「**沪深300ETF（510300.SS）** 上跑 **SMA 交叉**，快线 5 慢线 10，窗口要够均线算出来，至少 **两年**。」
- 「**BTC** 最近 **四个月**，RSI 策略，数据走 **Yahoo**；回测完给 Sharpe 和最大回撤。」
- 「在同一标的 **510300.SS**、同一区间上，用 **optimize** 只扫 RSI 的 `period` 取 **12 和 14** 两档，告诉我哪个 Sharpe 更高。」

### 5.3 Agent 回群模板（防截断）

在工具/`exec` 成功结束后，建议固定 **三段、每段不超过 5 行**：

1. **结论**：策略名 + 标的 + 区间 + 是否跑通（exit 0 / 非 0）。  
2. **关键指标**：从终端表格摘 **Total Return、Max Drawdown、Sharpe、Total Trades**（或 optimize 的 Top 行）；若用了 `--quiet`，可补一句「详见 `reports/` 下最新 `*_summary` / 终端完整表」。  
3. **数据与风险**：说明 `price_loader`（如 `china:tushare`、`cache:csv`、`yfinance`）；提醒样本量、过拟合、非实盘指令。

### 5.4 单条 `exec` 命令模板（复制后只改仓库根）

### 5.6 交互反例（工具链爆炸与模型退化）

**典型场景**：用户仅要「RSI 反转 + 510300 + 近一年 + 总收益/回撤/交易数/`price_loader`」，与 **§5.2 / §5.4** 完全一致；正确做法是 **一次** `exec` 跑 `backtest.py`（或 `run_backtest_trading_strategies.py`），从终端与 `data.attrs['price_loader']` 可得数据源。

**若 Agent 出现下列行为，属于跑偏，应立刻改 prompt 或 Agent 工具白名单：**

| 现象 | 问题本质 |
|------|-----------|
| 连续调用 `tool_fetch_etf_historical`、`tool_read_etf_daily`、`tool_fetch_etf_data`、**`python3 -c` 内联超长 JSON**、`tool_calculate_technical_indicators` 等 **十余次以上** | 用 **OpenClaw 采集/指标工具** 在对话里「手搓」一条研究回测，与 **`backtest.py` 内建 RSI 回测引擎** 重复；上下文暴涨、易触发限流与重复拉数。 |
| 内联 `exec` 塞整段 K 线进 `python3 -c` | 违反「单条、短命令」原则，易触发 **preflight 失败** 或 **length** 截断。 |
| 末尾出现 `TEAM???`、`I think I'm stuck`、大量 `?0?0?0?` | **模型在极长工具链后退化**（与钉钉「思考中」后乱码同类），结论**不可采信**。 |
| 最终编造「数据不足 35 条」「改 tool_fetch_stock_historical 拉 000300」等 | **未执行**或**未正确执行** §5.4 命令时的 **幻觉**；`price_loader` 应以 **`backtest.py` 打印的 `price_loader=`** 为准，不得凭想象写接口故障。 |

**纠偏（给值班 / Agent 维护者）**

1. 对该类需求 **只允许** 先跑 **§5.4 第一条 RSI 命令**；成功则按 **§5.3** 回三段。  
2. 若 `exec` 失败：只根据 **stderr 最后一行** 调整 `--period`、`--data-source` 或环境（见 **§6**），**不要**再叠加一串 `tool_fetch_*`。  
3. 若业务必须先走 MCP 拉数再回测：应在**工作流文档**里写清，与「仅 skill 回测」场景区分；钉钉闲聊默认走 **§5.4**。

## 6. 故障与排错

| 现象 | 原因 | 处理 |
|------|------|------|
| `exec preflight: complex interpreter invocation` | 使用 `cd ... && python3 ...` 等复合命令 | 改为**单条** `python3 /绝对路径/脚本.py ...` |
| `python: command not found` | 环境仅有 `python3` | 统一使用 `python3` |
| `Insufficient data... need at least 205` | SMA/EMA 等长 lookback + 窗口过短 | 加长 `--period` 或指定足够 `--start/--end` |
| `Error: China-market data fetch failed...`（auto/china 下 CN） | 本地管道失败且未开 Yahoo 兜底 | 修网络/依赖/缓存；或显式 `--data-source yfinance`；不推荐则设 `BACKTEST_CN_FALLBACK_YFINANCE=1` |
| `YFRateLimitError`（CN 标的误走 Yahoo） | 未使用默认 CN 管道 | 去掉强制 yfinance；保持 `data.provider: auto` 或省略 `--data-source` |
| 回复被截断、半句英文 | 多轮工具 + 长输出触顶 **`length`** | 采用 **§5.3** 短模板或拆两条消息；提高 Agent `maxTokens`；必要时重启 Gateway |
| `TEAM???`、大量 `?0?`、英文碎片 | 工具轮次过多 + 上下文爆炸，模型退化 | **停止追加工具**；改用 **§5.4 单条 exec**；见 **§5.6** |

## 8. 调参建议（Skill 侧）

- 策略级默认见 **`config/settings.yaml`** 中 **`strategies:`** 各小节。
- 单次回测覆盖默认：使用 **`--params`** 传 JSON，例如 `'{"fast_period":5,"slow_period":10}'`。
- 手续费与滑点：CLI **`--commission` / `--slippage`** 未写时取自 YAML **`backtest:`** 段；资金同理 **`--capital`**。

## 9. 生产执行建议

- 定时任务放在**数据采集完成之后**，减少半日缺口。
- 自动与人工使用**同一**命令模板与 `settings.yaml`，避免两套口径。
- 周期性归档报告文件名中的日期与参数，便于横向对比。
