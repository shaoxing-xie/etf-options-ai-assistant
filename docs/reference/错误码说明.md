# 工具错误码与执行日志

## 错误码表

`tool_runner.py` 返回的 JSON 中可能包含 `error_code` 字段，便于 Agent/工作流分支处理。

| error_code        | 含义       | 典型场景                         |
|-------------------|------------|----------------------------------|
| `VALIDATION_ERROR`| 参数校验失败 | Pydantic 校验未通过（如 Copilot 参数） |
| `UNKNOWN_TOOL`    | 未知工具   | 传入的工具名不在 TOOL_MAP 中     |
| `IMPORT_ERROR`    | 导入错误   | 工具模块或依赖加载失败           |
| `RUNTIME_ERROR`   | 执行异常   | 工具函数内部抛出异常            |

成功调用不会返回 `error_code`；失败时除 `error` 文案外会带 `error_code` 与必要时的 `details`。

## 工具执行日志（可选）

需排查耗时或失败时，可开启工具执行日志：

- **环境变量**：`OPTION_TRADING_ASSISTANT_LOG_TOOL_EXEC=1`（或 `true` / `yes`）
- **效果**：每次调用工具后，向 Python `logging` 输出一条：
  - `tool_exec <tool_name> duration_ms=<ms> success=true|false [error_code=...]`
- **用途**：与 OpenClaw 日志关联，查看耗时与是否成功。

## 清单与数据源

- 工具清单（名称、描述、参数）以 **config/tools_manifest.yaml** 为单一数据源。
- 修改清单后请执行：`python scripts/generate_tools_json.py`，生成 `config/tools_manifest.json` 供 OpenClaw 插件加载。
