# 工具参考手册（概述）

基于 OpenClaw 的 A股 / ETF 交易助手（含可选期权扩展能力），工具清单以 **config/tools_manifest.yaml** 为单一数据源，插件从 **config/tools_manifest.json**（由 `python scripts/generate_tools_json.py` 生成）加载并注册。

---

## 工具总数与分类

| 分类 | 数量 | 说明 |
|------|------|------|
| 数据采集 | 25 | 指数、ETF、期权、期货、个股、涨停、龙虎榜、资金流向、北向等 |
| 时段与编排 | 4 | A股时段、Copilot、可交易性过滤、事件哨兵 |
| 分析 | 16 | **技术指标**（`tool_calculate_technical_indicators`：`pandas_ta` standard / legacy、合并后配置 → `technical_indicators`，域文件：`config/domains/analytics.yaml`）、盘前/盘后/开盘分析、波动率、信号、风险评估、趋势、研究（Regime/轮动/策略）、**策略融合（`tool_strategy_engine`）** |
| 风险控制 | 5 | 仓位与硬锁定、止盈止损 |
| 策略效果跟踪 | 5 | 信号记录、表现、评分、权重 |
| 通知 | 5 | 飞书消息/卡片/信号/日报/风险预警 |
| 数据访问 | 6 | 从缓存读取指数/ETF/期权数据 |
| 回测 | 2 | 涨停回马枪回测、参数敏感性 |
| **合计** | **77** | 以 `config/tools_manifest.yaml` 为准，全部在 OpenClaw 中注册 |

> **说明**：上表分类数量为便于导航的概览；**权威工具列表与参数**以 **`config/tools_manifest.yaml`**（及生成的 `tools_manifest.json`）为准。

## 策略引擎与信号融合：`tool_strategy_engine`

- **用途**：在单路 `tool_generate_option_trading_signals`（别名 `tool_generate_signals`）之外，对多路 Rule 输出（如 src 期权信号 + ETF 趋势）做 **统一契约**（`SignalCandidate`）与 **加权融合**（v1 / v1.1 / v1.2）。以 **ETF 代码** 为 `symbol` 主键；ETF/A 股单路信号见 `tool_generate_etf_trading_signals` / `tool_generate_stock_trading_signals`（**未**默认并入 fusion v1）。  
- **参数与返回值**：见 `config/tools_manifest.yaml` 中 `tool_strategy_engine`（含「何时优先调用」长描述）。  
- **成功时 `data` 要点**：`candidates`、`fused`（主标的）、`fused_by_symbol`（多标的）、`summary`、`weights_effective`、`inputs_hash`（含策略版本与 policy/权重快照）、`policy_applied`、`underlyings` / `index_codes`。  
- **多 ETF**：`underlying` 与 `index_code` 英文逗号分隔且同序（如 `510300,510500` 与 `000300,000905`）。  
- **架构与 OpenClaw 调度**：`docs/architecture/strategy_engine_and_signal_fusion.md`、`config/openclaw_strategy_engine.yaml`、`plugins/strategy_engine/README.md`；步骤模板 `workflows/strategy_fusion_routine.yaml`；Skill `skills/ota-strategy-fusion-playbook/SKILL.md`。  
- **命令行试跑**（`tool_runner` 支持 `key=value` 多参数）：`python tool_runner.py tool_strategy_engine underlying=510300 index_code=000300`；多标的示例：`python tool_runner.py tool_strategy_engine underlying=510300,510500 index_code=000300,000905`

---

## 使用方式

1. **OpenClaw Agent**：在 Agent 配置中声明要调用的工具  
2. **OpenClaw 工作流**：在工作流中编排工具顺序与参数  
3. **Gateway API**：通过 OpenClaw Gateway 直接调用  

工具名以 `tool_` 前缀开头，参数为 JSON。
