# 使用场景与最佳实践

## 使用场景示例

### 场景1：盘前分析流程（9:15执行）

```yaml
tasks:
  - name: morning_data_collection
    schedule: "15 9 * * 1-5"
    tools:
      - name: tool_fetch_index_opening
      - name: tool_fetch_etf_realtime
        params: { etf_code: "510300,510050" }
      - name: tool_fetch_global_index_spot

  - name: pre_market_analysis
    schedule: "15 9 * * 1-5"
    tools:
      - name: tool_analyze_before_open
      - name: tool_calculate_technical_indicators
        params: { symbol: "510300", data_type: "etf_daily", indicators: ["RSI", "MACD", "MA"] }
      - name: tool_predict_intraday_range
        params: { underlying: "510300" }
      - name: tool_send_feishu_message
        params: { title: "盘前分析报告", message_type: "rich_text" }
```

## 最佳实践

### 1. 工具调用顺序

**数据采集 → 分析 → 决策 → 风险控制 → 通知**

1. tool_fetch_etf_realtime  
2. tool_calculate_technical_indicators  
3. tool_check_etf_index_consistency  
4. tool_generate_trend_following_signal  
5. tool_calculate_position_size  
6. tool_calculate_stop_loss_take_profit  
7. tool_send_signal_alert  

### 2. 参数建议

- **技术指标**：日线 `lookback_days: 120`，分钟 `lookback_days: 5`，常用 `["RSI", "MACD", "MA", "ATR"]`
- **趋势跟踪**：`lookback_days: 60`，`min_trend_strength: 0.6`，`min_consistency_score: 0.7`
- **风险控制**：`hard_limit_pct: 0.02`（2%），止损约 1–2%，止盈约 2–3%

### 3. 错误处理

- 检查返回中的 `success` 字段；失败时查看 `error`、`message` 及可选 `error_code`
- 错误码说明见 [错误码说明.md](错误码说明.md)

### 4. 数据缓存

- 实时数据不缓存；历史/日线优先读缓存；分钟数据缓存最近 5 天

### 5. 性能

- 批量：用逗号分隔多标的
- 优先用数据访问类工具读缓存
- 工作流中可并行执行
- 定时任务避免过于频繁

## 代码与时间格式

- **指数**：000001、000300、000016（上证）；399001、399006、399005（深证）
- **ETF**：510300、510050、159915
- **期权合约**：标的+行权价+C/P+到期月
- **日期**：YYYYMMDD；**时间**：YYYY-MM-DD HH:MM:SS
- **交易时间**：上午 09:30–11:30，下午 13:00–15:00；集合竞价 09:15–09:25、14:57–15:00
