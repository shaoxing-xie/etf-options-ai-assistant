## 1. 环境变量与配置入口（摘要）

| 类别 | 变量或文件 | 用途 |
|------|------------|------|
| 通知 | `FEISHU_WEBHOOK_URL`、`FEISHU_APP_ID`、`FEISHU_APP_SECRET` | 合并后配置 → `notification`（`${...}` 占位；域文件：`config/domains/outbound.yaml`） |
| 策略融合权重 | `STRATEGY_FUSION_WEIGHTS_PATH`（可选） | 覆盖 `data/strategy_fusion_effective_weights.json` |
| 工程根 | `ETF_OPTIONS_PROJECT`（部分脚本） | 指向本仓库根目录 |
| 主配置 | 合并后配置（分层 YAML） | 市场、存储 `data_storage.data_dir`、日志、`prediction_quality` 等（入口见 `docs/configuration/README.md`） |
| OpenClaw 策略引擎 | `config/openclaw_strategy_engine.yaml` | 路由、进化、权重路径 |
| 工具清单源 | `config/tools_manifest.yaml` | 修改后执行 `python3 scripts/generate_tools_json.py` 生成 `config/tools_manifest.json` |

更全的部署变量见 `docs/publish/` 与 `docs/getting-started/`。

## 2. OpenClaw 工具（本仓 `tools_manifest`）

以下 **tool id** 由本仓库 **`option-trading-assistant`** 经 `tool_runner.py` 暴露（与 Gateway manifest 对齐）。**行情类 `tool_fetch_*` / 统一抓取** 主要在 **`openclaw-data-china-stock`** 扩展中注册；本表以本仓清单为准，详见清单顶部注释。

**验收（通用）**（在仓库根、已激活 venv）：

```bash
python tool_runner.py <tool_id> key=value ...
```

具体参数见 [`../reference/工具参考手册.md`](../reference/工具参考手册.md) 或 `config/tools_manifest.yaml`。

## 3. 工作流 YAML（`workflows/`）

调度与用途的**权威表格**见 [`workflows/README.md`](../../workflows/README.md) 一览表。本节仅作索引分组。

| 分组 | 示例文件 |
|------|----------|
| 交易日主链路 | `before_open_analysis.yaml`、`opening_analysis.yaml`、`intraday_analysis.yaml`、`after_close_analysis_enhanced.yaml` |
| 信号 / 巡检 | `signal_generation.yaml`、`signal_generation_on_demand.yaml`、`signal_risk_inspection.yaml`、`etf_510300_intraday_monitor.yaml` |
| 报告 / 研究 | `daily_market_report.yaml`、`limitup_pullback_after_close.yaml`、`etf_rotation_research.yaml`、`strategy_research.yaml`、`*_agent.yaml` |
| 策略运维 | `strategy_evaluation.yaml`、`strategy_weight_adjustment.yaml`、`strategy_fusion_routine.yaml` |
| 预测闭环 | `prediction_verification.yaml`（另见脚本 `scripts/prediction_metrics_weekly.py`） |
| 进化 / CI | `*_evolution_on_demand.yaml`、`ci_autofix_triage_on_demand.yaml`、`cron_error_autofix_on_demand.yaml`、`quality_backstop_audit.yaml` |

**Cron 真源**：以本机 `~/.openclaw/cron/jobs.json` 为准；YAML 内 `schedule` 为设计参考。

**校验**：`python scripts/validate_workflows.py` 将 `workflows/*.yaml` 中的 `tool:` 与 `config/tools_manifest.yaml` ∪ `config/workflow_external_tool_ids.txt`（扩展注册的 fetch/read 类 id）比对；CI 可调用 `tests/integration/run_all_workflow_tests.py`。

## 5. 数据路径速查

与读写方、稳定性策略见 **[跨插件数据契约.md](./跨插件数据契约.md)**。常用相对路径（相对仓库根）：

- 行情 parquet：`data/cache/**`
- 融合权重：`data/strategy_fusion_effective_weights.json`
- 信号与预测：`data/signal_records/`、`data/prediction_records/`
- 主日志：`logs/option_trading_{date}.log`（模板来自合并后配置 `system.logging`，域文件：`config/domains/platform.yaml`）
