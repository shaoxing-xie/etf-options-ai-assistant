# WORKFLOW_AUTO.md - 自动启动工作流

此文件定义了 Agent 在会话启动时自动执行的工作流。在上下文压缩后，Agent 应该自动读取此文件以恢复操作协议。

## 🚀 启动时自动执行

### 1. 会话初始化流程

每次新会话开始时，Agent 应该：

1. **读取身份文件**（按顺序）：
   - `SOUL.md` - Agent 的身份和核心特性
   - `USER.md` - 用户的身份和偏好
   - `MEMORY.md` - 长期记忆和交易规则
   - `memory/YYYY-MM-DD.md` - 今天的记忆文件（如果存在）
   - `memory/YYYY-MM-DD.md` - 昨天的记忆文件（如果存在）

2. **检查工作流状态**：
   - 验证 Cron 任务是否正常运行
   - 检查工具是否可用（包括 `web_search` 在内的工具均可按需使用）
   - 确认通知渠道（飞书等）是否配置正确

### 2. 主会话 (Main Session) 特殊流程

如果是在主会话（直接与用户对话）：

1. **必须读取**：
   - `MEMORY.md` - 长期记忆
   - `memory/` 目录下最近的日期文件

2. **检查系统状态**：
   - API 速率限制状态
   - 模型可用性
   - 工具注册状态

### 3. Cron 会话流程

如果是在 Cron 触发的会话中：

1. **读取相关配置**：
   - `WORKFLOW_CONFIG.md` - 工作流配置说明
   - 相关的 memory 文件（根据任务类型）

2. **执行任务**：
   - 按照 Cron 任务配置的消息执行
   - 使用相应的工具完成任务
   - 发送结果到通知渠道

## ✅ 关键操作规则（避免答错）

### 当用户问「列出定时任务 / cron 有哪些」

1. **先查 OpenClaw 自带 Cron**（这是 OpenClaw 真正使用的定时任务系统）：
   - 文件：`~/.openclaw/cron/jobs.json`
   - 或命令：`openclaw cron list`
2. **只有当用户明确问“crontab/系统定时任务”时，才去执行** `crontab -l`。
3. 如果系统 `crontab` 为空，但 OpenClaw Cron 有任务，必须明确说明两者是不同系统：
   - “系统 crontab 没有任务”
   - “OpenClaw Cron 已配置 X 个任务（从 jobs.json / openclaw cron list 获取）”

## ⚠️ 注意事项

1. **API 速率限制**：
   - 如果遇到 API 速率限制，应该：
     - 等待一段时间后重试
     - 或切换到备用模型
     - 或使用本地模型

2. **工具可用性**：
   - `web_search` 工具已启用，可在需要最新/外部信息时按需调用
   - 波动率预测工具仅在交易日可用
   - 飞书消息使用 text 模式更稳定

3. **文件读取**：
   - 如果文件不存在，应该创建或跳过
   - 如果文件格式错误，应该报告但继续执行

## 🔄 压缩后恢复

在上下文压缩后，Agent 应该：

1. **立即读取**：
   - `WORKFLOW_AUTO.md`（本文件）
   - `memory/YYYY-MM-DD.md`（最近的日期文件）

2. **恢复状态**：
   - 重新加载必要的配置
   - 恢复工作流状态
   - 确认工具可用性

3. **继续执行**：
   - 按照正常流程继续工作
   - 不要重复已完成的步骤
