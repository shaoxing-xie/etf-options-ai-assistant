# 跨插件数据契约（阶段 1）

> **目的**：固定「谁写谁读、路径如何解析、哪些 JSON 视为稳定契约」，避免主仓 `option-trading-assistant` 与 **`openclaw-data-china-stock`**（行情采集）及 OpenClaw 运行时之间的隐式耦合失控。  
> **关联**：[`能力地图.md`](./能力地图.md)、[`模块化改造策略-插件与Skill分层.md`](./模块化改造策略-插件与Skill分层.md)、[`../architecture/strategy_engine_and_signal_fusion.md`](../architecture/strategy_engine_and_signal_fusion.md)。

---

## 1. 工程根路径（所有相对路径的基准）

| 约定 | 说明 |
|------|------|
| **仓库根** | 含 `tool_runner.py`、`config/`、`data/` 的目录；OpenClaw 插件通过 `index.ts` 调用 `tool_runner.py` 时，工作目录应与此根一致（见部署脚本与 `plugins.load.paths`）。 |
| **`ETF_OPTIONS_PROJECT`** | 部分运维脚本（如监控）用该环境变量指向仓库根；未设置时以脚本内推导或当前工作目录为准。 |
| **Python 路径** | `tool_runner.py`、插件内 `import` 依赖 `PYTHONPATH` 含仓库根（安装脚本通常已配置）。 |

缓存与数据文件的 **相对路径**（如 `data/cache/...`）均相对于上述仓库根解析，除非配置中显式给出绝对路径。

---

## 2. 行情缓存目录（主仓 ↔ 外置采集插件）

| 项 | 契约 |
|----|------|
| **逻辑根** | 合并后配置 → `system.data_storage.data_dir`（默认 **`data`**；域文件：`config/domains/platform.yaml`）下的 **`data/cache/`**。 |
| **布局** | 由 `src/data_cache.get_cache_file_path` 定义，例如：`data/cache/etf_daily/{symbol}/{date}.parquet`、`data/cache/index_minute/...` 等。 |
| **写入方** | 采集侧：`plugins/data_collection`（符号链接至 **`openclaw-data-china-stock`** 内实现）、`src/data_collector` 回填逻辑、`read_cache_data` 触发的补数路径等。 |
| **读取方** | `plugins/data_access.read_cache_data` / `merged.read_market_data`、各分析工具、依赖「读缓存」的工作流（见 `workflows/README.md`）。 |
| **不变量** | 同一 `data_type` + `symbol` + `date`（+ 可选 `period`）对应的路径规则不变；若变更目录结构，须同步改 `data_cache.py` 与所有依赖扫描路径的脚本，并在 CHANGELOG 标明 **破坏性变更**。 |

外置插件仓库内若有独立 `data/` 或工具级缓存，仅当 **运行配置仍指向本仓库 `data/`** 时与主契约一致；多机部署时以 **实际 `tool_runner`  cwd 与 config** 为准。

---

## 3. 策略融合有效权重文件

| 项 | 契约 |
|----|------|
| **默认路径** | `data/strategy_fusion_effective_weights.json`（相对仓库根）。 |
| **配置覆盖** | `config/openclaw_strategy_engine.yaml` → `evolution.effective_weights_path`。 |
| **环境变量覆盖** | `STRATEGY_FUSION_WEIGHTS_PATH`：绝对路径，用于测试/沙箱（见 `plugins/strategy_engine/README.md`、`plugins/analysis/strategy_weight_manager.py`）。 |
| **写入方** | `tool_strategy_engine`：当 `evolution.persist_effective_weights: true` 时，将本次 **`weights_effective`** 落盘。 |
| **读取方** | `tool_strategy_engine`（`use_dynamic_weights=true` 时优先读此文件）；`get_strategy_weights` / `tool_get_strategy_weights` 等与 `strategy_id` 键对齐的权重逻辑。 |
| **键语义** | JSON 内键须与 `config/strategy_fusion.yaml` 中 `strategy_weights` 及 **`SignalCandidate.strategy_id`** 一致；新增策略时须同时更新融合配置与可选的 `evolution.fusion_weight_keys`。 |

## 4. 其他高价值运行时文件（读写关系摘要）

以下路径被多个模块依赖，变更字段语义前需评估消费者。

| 路径 | 主要写入方 | 主要读取方 | 稳定性说明 |
|------|------------|------------|------------|
| `data/signal_records/` | 信号记录工具、工作流 | 回测、评估、巡检 | DB/SQLite + JSON；列与表结构变更需迁移说明。 |
| `data/prediction_records/predictions_*.json` | `prediction_recorder`、预测类工具 | `prediction_verification` 工作流、周报脚本 | 字段 `range_pct`、`confidence`、`actual_range`、`verified` 等见 `docs/openclaw/宽基ETF巡检快报-日内波动区间收敛说明.md`。 |
| `data/volatility_ranges/` | 波动预测工具链 | 分析、进化工作流 | 与预测记录口径对齐；收敛规则变更需同步文档。 |
| `data/trend_analysis/` | `save_trend_analysis` / 趋势三工具 | 报告、复盘、`load_trend_analysis` | 子目录：`after_close/`、`before_open/`、**`opening/`**（与 `before_open` 分离）；单文件 `{YYYYMMDD}.json`；盘前可读 `after_close` 落盘复用盘后结论（见 `analyze_market_before_open`）。 |
| `data/cache/`（非 parquet） | LLM 上下文、广度等 | 同名读取逻辑 | 与 `PROJECT_LAYOUT` 中「市场广度缓存」等描述一致。 |
| `logs/option_trading_{date}.log` | 全局日志配置 | 排障、 observability | 路径由合并后配置 → `system.logging.file_path` 模板决定（域文件：`config/domains/platform.yaml`）。 |

## 5. JSON / 结构化输出的稳定性策略

1. **工具对外 JSON**（OpenClaw schema）：在 `config/tools_manifest.yaml` 中登记的工具，其返回结构被 Agent 与工作流依赖；删除字段、改类型视为 **破坏性变更**，应升级文档与调用方，并在 CHANGELOG 标注。  
2. **内部契约**：`SignalCandidate` / `FusionResult` 见 `plugins/strategy_engine/schemas.py` 与架构文档；改字段须同步 `tool_strategy_engine` 消费者与测试 `tests/test_strategy_engine.py`。  
3. **外置插件 DTO**：`openclaw-data-china-stock` 内 LEVEL/DTO 变更以该仓库 README、ROADMAP 为准；主仓侧若解析其输出，须在集成测试中锁定关键字段。
